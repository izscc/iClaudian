## 1. 背景与动机

iClaudian 是 `YishenTu/claudian` 的分叉(fork 点为上游 tag `2.0.10`),已显著扩展:在上游的 Claude + Codex 之上新增了 `acp / antigravity / copilot / gemini / opencode` 等 provider,以及提示词预置等功能。

自 fork 以来:
- 上游 claudian 已推进到 `upstream/main`(2.0.18 之后),期间 **37 个 commit(26 个非发布)**,含多项 bug 修复与功能。
- **Claude Opus 4.8 已发布**,系统 Claude Code CLI 已是 `2.1.158`。
- `@anthropic-ai/claude-agent-sdk` 本地为 `0.2.112`,npm 最新 `0.3.158`。
- Copilot CLI(`1.0.56`)新增了 `claude-opus-4.8` 模型。
- Antigravity CLI(`agy 1.0.3`)模型与 flag 未变。

本次发布的目标是把这些外部演进同步进 iClaudian,并发版 2.3.0。

### 1.1 关键事实(已核验)

> 以下结论均经实测/读源码/查官方文档核验,是设计的基础。

1. **Opus 4.8 已能用,无需为它升级 SDK**。iClaudian 通过 `options.pathToClaudeCodeExecutable = ctx.cliPath` 走**系统 CLI**(`2.1.158`),绕过 SDK 自带 `cli.js`。系统 CLI 已认识 `claude-opus-4-8`、`xhigh`、`max`、adaptive thinking。iClaudian 发送家族别名 `opus`/`opus[1m]`,CLI 自动解析到最新 Opus(即 4.8)。
2. **不存在需要新增的 API 思考等级**。4.8 的 API effort 枚举即 `low|medium|high|xhigh|max`,与 iClaudian 现有 `EffortLevel` 完全一致;`xhigh` 与 `max` 在 4.8 上均可用。`ultracode`(= `xhigh` + 自动 workflow 授权)与 `/fast`(交互式快速模式)均为 Claude Code 交互层概念,**SDK `Options` 无对应可编程字段**,本次不纳入。
3. **思考显示确实是当前回归**。SDK 类型 `ThinkingAdaptive = { type: 'adaptive'; display?: 'summarized' | 'omitted' }`。iClaudian 现在只设 `{ type: 'adaptive' }`,未传 `display`;4.7/4.8 的 API 默认 `display` 为 `omitted`(4.6 曾是 `summarized`),导致思考过程文本在面板里为空。该字段在本地 `0.2.112` SDK 中就已存在,修复**不依赖**升级。
4. **`supportsXHighEffort` 与 `isAdaptiveThinkingModel` 的正则已匹配 `claude-opus-4-8`**(`/claude-opus-(4-[7-9]|[5-9])/`),无需改正则。
5. **SDK 0.2→0.3 唯一强制代码触点**是 transcript 解析:headless/SDK 会话改用 `TaskCreate/TaskUpdate/TaskGet/TaskList` 取代 `TodoWrite`(0.2.136 起弃用)。另需把 `@anthropic-ai/sdk` 与 `zod` 提升为直接依赖(0.3.x 将其 peerDeps 化)。thinking/effort/spawn/permission 路径**无破坏性变更**。

## 2. 目标与非目标

### 目标
- G1:修复 Opus 4.7/4.8 思考过程文本不显示的回归。
- G2:把 Opus(及 1M 变体)的默认 effort 提到 `xhigh`,贴合 Anthropic 对编码/agent 场景的推荐默认。

- G5:从上游均衡移植高价值 bug 修复与功能。
- G6:更新文档(README、过时的 CLAUDE.md),发版 2.3.0。

- N1:**不**在 Claude 模型下拉中新增显式 `Opus 4.8` 条目(`opus` 别名已路由到 4.8)。
- N2:**不**支持 `ultracode` / `/fast`(SDK 无可编程接口)。
- N3:Antigravity **不**做强制代码改动(已与 agy 1.0.3 对齐)。
- N4:**不**移植 `4cefc86`(改名 realclaudian)、`6ae647c`(Codex 协议已分叉)、`00c1c33`(抬高 Obsidian 最低版本)。

### A. Opus 4.8 思考体验(Claude provider)

#### A1 — 思考显示修正(修复回归)
- **改动**:`src/providers/claude/runtime/ClaudeQueryOptionsBuilder.ts` 的 `applyThinking()`(约 L296–314)。
  - 现:`options.thinking = { type: 'adaptive' }`(L303)。
  - 改:`options.thinking = { type: 'adaptive', display: 'summarized' }`。
  - 两条构建路径都经过 `applyThinking()`(`buildPersistentQueryOptions` L153、`buildColdStartQueryOptions` L206),一处改动即覆盖。
- **动态路径核查**:`src/providers/claude/runtime/ClaudeDynamicUpdates.ts`(`setModel` L71、`applyFlagSettings({effortLevel})` L104)。需确认会话中途切换模型(如 4.6→4.8)时 thinking 配置是否需重新下发;若 `setModel` 不重置 thinking,则当前 `display` 配置仍生效(可接受);若发现思考在动态切模型后丢失,补一次 thinking 重设。
- **渲染侧无需改**:`src/providers/claude/stream/transformClaudeMessage.ts`(L368/488/506-507)已读取 `block.thinking` / `event.content_block.thinking` / `event.delta.thinking`,只要上游 emit 非空即可渲染。
- **决策**:`display:'summarized'` **无条件**用于 adaptive 路径——对 4.6 无害(本就 summarized),对 4.7/4.8 修复空白。本次不为它加设置项。
- **TDD**:`tests/unit/providers/claude/runtime/QueryOptionsBuilder.test.ts` 增断言:adaptive 路径下 `options.thinking` 含 `display:'summarized'`;非自适应路径仍走 `maxThinkingTokens` 不变。
- **验收**:在真实 Opus 4.8 会话中,思考面板能渲染 summarized 思考文本(对照 4.6 行为无回归)。

#### A2 — 默认 effort 提到 xhigh
- **改动**:`src/providers/claude/types/models.ts` 的 `DEFAULT_EFFORT_LEVEL`(L38–44):`opus` 与 `opus[1m]` 由 `'high'` → `'xhigh'`。`haiku`/`sonnet`/`sonnet[1m]` 保持 `'high'`。
- **语义**:`applyModelDefaults`(`ClaudeChatUIConfig.ts` L72–87)在**切换到该模型时**写入默认 effort;已保存 `effortLevel` 的用户设置不被覆盖。`supportsXHighEffort` 已允许 opus 的 `xhigh`,UI `getReasoningOptions`(L47–55)会正确展示。
- **权衡**:xhigh 比 high 更费 token——已与用户确认接受。
- **TDD**:`tests/unit/providers/claude/types/types.test.ts` 与 `tests/unit/providers/claude/ui/ClaudeChatUIConfig.test.ts` 更新 `DEFAULT_EFFORT_LEVEL` / `applyModelDefaults` 的期望(opus 默认 = `xhigh`)。
- **验收**:新建会话选 Opus,effort 默认显示为 XHigh;Haiku/Sonnet 仍为 High。

### D. SDK 0.2.112 → 0.3.158

#### D1 — 依赖升级

#### D2 — transcript 解析迁移(唯一强制代码触点)
- **背景**:0.2.136 起 headless/SDK 会话弃用 `TodoWrite`,改用 `Task*` 工具。升级后新会话不再 emit `TodoWrite`,iClaudian 的 todo 面板解析会变空。
- **改动**:

  - `src/core/tools/todo.ts`:解析逻辑在识别 `TodoWrite` 之外,同时识别 `Task*` 工具的输入/输出,映射到现有 todo 面板数据模型(保持对旧 `TodoWrite` 会话的向后兼容)。
- **TDD**:为 `todo.ts` 增测试:`TaskCreate/TaskUpdate` 事件能正确填充 todo 面板;旧 `TodoWrite` 事件仍可解析。
- **核查**:确认 iClaudian 无 `unstable_v2_*` 用法(已核验:`SDKSession*` 是本地 `sdkHistoryTypes`,非被移除的 SDK 符号),v2 session API 移除**不影响**本项目。
